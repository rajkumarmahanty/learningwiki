🚀 Intelligent AIOps Ecosystem v2 V2 Live

Autonomous Predictive Self-Healing Infrastructure Platform

This project transitions infrastructure management from reactive firefighting to proactive, AI-driven remediation using RAG (Retrieval-Augmented Generation) and LLM reasoning.

🏛️ System Architecture

The ecosystem leverages a multi-layered approach to identify and fix anomalies before they impact end users.

Layer Stack Primary Role
Observability Prometheus Trend analysis and linear regression for predictive alerting.
Orchestration n8n The central workflow "brain" connecting webhooks to agents.
Reasoning GPT-4o + RAG Analyzing live Kubernetes logs against historical incident runbooks.
Memory Pinecone Vector storage for long-term engineering knowledge and SOPs.

📁 Folder Structure

INTELLIGENT_AIOPS_V2/
├── .github/workflows/   # DevSecOps Pipeline (Snyk SCA/IaC)
├── agent_tools/         # K8s Python SDK Wrappers (Patching)
├── n8n_workflows/       # Workflow Logic JSON Exports
├── scripts/             # RAG Seeding & Webhook Test Simulators
├── langchain_agent.py   # AI Core (Reasoning & Log Processing)
└── docker-compose.yaml  # Local Infrastructure Stack

📄 Component Explanation

  • langchain_agent.py: The primary script that creates the context window for GPT-4o.
  • state_manager.py: Manages the short-term state in Redis for Slack approval tokens.
  • upsert_rag_data.py: Converts PDF/Text runbooks into vectors for Pinecone.

🛠️ Execution Steps

1. Environment Setup

Populate your .env with API keys and boot the local stack:

docker-compose up --build -d

2. Seed the Knowledge Base

Inject engineering expertise into the Vector Database:

python scripts/upsert_rag_data.py

3. Initiate the "Fire Drill"

Trigger a simulated predictive alert to test the Slack approval flow:

python scripts/test_webhook.py
