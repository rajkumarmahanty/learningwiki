Terraform + GitHub Actions CI/CD

Automating Infrastructure as Code (IaC) with secure pipelines.

Workflow Overview

A professional Terraform pipeline automates testing and deployment based on GitHub events.

  • Pull Request: Runs terraform plan to preview changes.
  • Push to Main: Runs terraform apply to deploy to the Cloud.
Pro Tip: In 2026, use OIDC (OpenID Connect) instead of long-lived AWS Access Keys for better security.

The Workflow File

Create this file at .github/workflows/terraform.yml:

name: "Terraform Deployment"

on:
  push:
    branches: [ "main" ]
  pull_request:
    branches: [ "main" ]

permissions:
  id-token: write
  contents: read
  pull-requests: write

jobs:
  terraform:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Setup Terraform
        uses: hashicorp/setup-terraform@v3

      - name: Terraform Init
        run: terraform init

      - name: Terraform Plan
        if: github.event_name == 'pull_request'
        run: terraform plan

      - name: Terraform Apply
        if: github.ref == 'refs/heads/main' && github.event_name == 'push'
        run: terraform apply -auto-approve

Critical Checklist

  1. Remote Backend: Ensure your terraform.tfstate is stored in S3/GCS/Azure Blob. GitHub runners are ephemeral!
  2. Secrets: Store sensitive data in GitHub Secrets (Settings > Secrets and variables > Actions).
  3. State Locking: Use DynamoDB (for AWS) to prevent concurrent runs from corrupting your state.
